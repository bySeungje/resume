## 6. 킬러 콘텐츠 TOP 3 (10/10 에이전트 합의)

### 1위: COBOL 40년 이자계산 오류 발견 — 70억 원 소급 보정 (8/10 선정)

> "COBOL 반올림 로직이 전기공급약관 기준과 불일치 → 건당 1원의 systematic bias → 10만+ 고객 × 수십 년 = 70억 원+ → 한전+KDN 합동 시연회에서 데이터로 증명 → 소급 보정"

**Agent 4 판정: Strong Pass** — "수백 회 면접 중 상위 5% 사례"
**Agent 1 한마디:** "코드 한 줄의 반올림 오류에서 70억을 발견하는 눈은 Redis 경험 3년보다 귀하다"

### 2위: Spring Batch 마이그레이션 — 월 175만 건 80%+ 단축 (7/10 선정)

> "3축 개선: ① 1,000건 청크+8스레드 ② 한전 지역본부 6개=파티셔닝 키 ③ 순차 JOB→독립 Step 병렬 Flow. PL로서 5명(주니어3+6년차2) 교육"

**면접 킬러 멘트:** "기술만으로 8배가 아니라, 한전 지역본부 구조를 파티셔닝 키로 쓴 건 도메인을 아는 사람만 할 수 있는 설계입니다."

### 3위: 외부 마켓 3사 공통 아키텍처 — 연동 92% 단축 (6/10 선정)

> "eBay 5개월→11번가 2~3주. Anti-Corruption Layer로 외부 주문을 Canonical Data Model에 매핑, Adapter Pattern으로 마켓별 격리. 매출 ~20% 증가"

**면접 킬러 멘트:** "5개월이 '기술 난이도'가 아니라 '구조적 설계 부재'였다는 진단이 출발점이었습니다."

## 8. "결정적 1가지" — payment-api-demo (10/10 합의 유지)

**8/10 에이전트가 "이것 하나가 판을 바꾼다"고 재확인**

```
payment-api-demo/
├── src/main/java/
│   ├── PaymentService.java        # 결제 처리 + 상태 머신
│   ├── SettlementBatch.java       # 파티셔닝 정산 배치
│   ├── MarketplaceAdapter.java    # ACL + Adapter Pattern
│   └── config/RedisConfig.java    # 캐싱 + 분산 락
├── src/test/java/                 # JUnit5 + Mockito (커버리지 70%+)
├── docker-compose.yml             # Redis + PostgreSQL + App + Prometheus + Grafana
├── docs/ADR/                      # 아키텍처 의사결정 기록
├── README.md                      # 아키텍처 다이어그램 + 설계 근거
└── .github/workflows/ci.yml      # GitHub Actions CI
```

**이 프로젝트가 한 번에 해소하는 갭:**
- Redis (캐싱, 분산 락) → Deal-breaker 해소
- JUnit5/Mockito 테스트 → 서비스 기업 문화 적합성
- Docker Compose → 모던 인프라 이해도
- 도메인 경험의 코드화 → "말만 하는 사람"이 아닌 증거
- Prometheus/Grafana → 모니터링 기본 경험
