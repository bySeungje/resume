## 2. 소개 (About)

**Before:** 기술 나열 + 성과 병렬 배치
**After:** _(Agent 2: 임팩트 중심 / Agent 9: 경험 언어화 / Agent 8: 희소성 / 킬러 콘텐츠 TOP 3 순서)_

```
11년간 금융·공공 도메인에서 "문제를 정의하고, 시스템으로 해결하는" 백엔드 개발자입니다.
AML(자금세탁방지) + 마이데이터 + 결제정산 — 세 영역을 모두 실무 경험한 개발자는 국내 시장 3~5%입니다.

한전KDN 3회, 우체국금융 2회 재투입 — 회사가 아니라 저를 지명해서 요청하는 개발자입니다.

━━ 킬러 성과 3가지 ━━

1. COBOL 40년 이자계산 오류 발견 · 70억 원 누적 오차 소급 보정
   한전KDN 정산 시스템 데이터를 분석하던 중, 원금+이자 ≠ 납부예정금액인 불일치를 발견했습니다. COBOL 코드의 반올림 처리가 전기공급약관 기준과 다르게 구현되어 있었고, 건당 1원의 체계적 편향(systematic bias)이 40년간 한 방향으로만 누적되어 70억 원 이상의 오차를 만들었습니다. 한전과 KDN 직원이 모인 시연회에서 운영 데이터로 직접 증명하여 소급 보정을 실행했습니다.

2. Spring Batch 마이그레이션 — 월 175만 건 처리 시간 80%+ 단축
   3가지 축으로 개선했습니다. 첫째, COBOL의 건별 순차 처리를 1,000건 청크 + 8스레드로 전환. 둘째, 한전 지역본부 6개 구조를 파티셔닝 키로 활용한 데이터 분할 병렬 처리. 셋째, 순차 실행 JOB 체인을 독립 Step 병렬 Flow로 재설계. PL로서 Spring Batch 미경험 개발자 5명(주니어 3 + 6년차 2)을 교육하여 독립 개발 가능 수준까지 이끌었습니다.

3. 외부 마켓 3사 공통 아키텍처 → 연동 리드타임 92% 단축
   eBay 첫 연동 5개월의 원인이 '기술 난이도'가 아니라 '구조적 설계 부재'임을 진단했습니다. Anti-Corruption Layer로 외부 마켓의 주문·클레임 구조를 내부 표준 모델(Canonical Data Model)에 매핑하고, 마켓별 차이를 Adapter Pattern으로 격리. 새 마켓 추가 시 Adapter만 구현하면 결제·취소·환불 비즈니스 로직이 전부 재사용되어 11번가는 2~3주 만에 연동 완료. 매출 ~20% 증가에 기여.

━━ 도메인 전문성 ━━

• 금융 규제: AML STR 76개 룰 병렬화(4중 중첩→룰 단위 프로시저, 5시간→1시간 48분), 마이데이터 2.0 5개 금융 권역 API 설계
• 대규모 배치: 월 175만 건 정산, 일 17K~26K 주문 결제/클레임 2년 운영, 전일자/당일 이중 시점 데이터 관리
• 레거시 전환: COBOL → Spring Batch, 40년 데이터 정합성 보장하며 현대화, 단건→대량 비교→병행 운영 3단계 검증
• 0→1 서비스: CTO 2회, MeowRo 앱 1인 완성(마케팅 0원, 다운로드 432건, 창원시 B2G 예산 3,000만원 확보, NTIS 유사도 0점)

기술 블로그: https://velog.io/@byseungje
GitHub: https://github.com/byseungje
```

> **v3.0 변경 이유:**
> - COBOL 오류: "COBOL vs Java"를 **"COBOL 반올림 로직 자체가 약관 기준과 불일치"**로 사실 교정 + **70억 원** 규모 명시 + 시연회 에피소드 추가
